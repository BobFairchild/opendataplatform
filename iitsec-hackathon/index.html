The Open Data Platform (ODP) is designed to aid in rapidly setting up a Hadoop environment and facilitate the ingestion of data to allow data analysis and visualization.

Ingestion

The ODP project for data ingestion utilizes Apache NiFi. There are some example templates that demonstrate the ability of the platform to process incoming data as a REST API from spending-api.us or from CSV files. The example use case seeks to utilize geocoder to extract latitude/longitude coordinates based off zipcodes and transform the data into JSON to be indexed into ElasticSearch and then use Kibana as a visualization into that data.

The NiFi Templates are just guidelines and there are many default processors that come packaged with Nifi that allow the user to route and transform data into the system.

This exposes an empty NiFi canvas where you can use the component toolbar to drag existing templates onto.

Saving Nifi Templates

1. In the operate window on the left hand side of the screen, select the “Create Template” icon.
2. To download it, click the icon in the upper right hand corner, and on the menu that pops up, select the Templates icon.
3. Then find the template you want and click the icon to download.

Uploading and Instantiating the Sailfish Template

1. In the operate window on the left hand side of the screen, select “Upload Template”
2. In the menu at the top, click the template icon and drag it onto the canvas. Then select the template to add it to the canvas.
3. In the operate window on the left hand side of the screen, select the “Configuration” icon.
4. On the Controller Services tab, you need to edit the StandardSSLContextService
    a. Set the password “sailfish” for the keystore and truststore password fields.

You can configure the run by providing the parameters to the first processor in the flow, “Setup Config” processor. Right click on processor, then select properties tab. You can then change the config values for:

  • email: email address for sailfish
  • password: password for sailfish
  • sailfish_hostname: hostname for the sailfish server
  • sailfish_catch_id: id for the sailfish catch to ingest
  • sailfish_net_id: sailfish net id

Flexibility

The templates shown here can be customized and other processors to feed data into any number of other solutions such as HDFS, HBase, Hive, Solr, etc. It is up to the user to determine the best way they want to load and analyze the data.
